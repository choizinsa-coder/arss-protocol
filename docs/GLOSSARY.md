## 1. 사업 4단계

| 단계 | 정의 | 게이트 |
|------|------|--------|
| 사업 정의 | 무엇을 누구에게 팔 것인지 결정 | 없음 (준비) |
| 사업 준비 | 기능 설계, 고객 탐색, 시범 대상 확보 | 없음 (준비) |
| 시범 운영(Pilot) | 제한된 실제 환경에서 검증 | 비오님 승인 필요 |
| 사업 착수 | 실제 고객에게 서비스를 제공하고 운영 시작 | 안정 1단계 완성 + 비오님 승인 |

S416 로드맵의 "비즈니스 착수 금지"는 위 표의 **사업 착수**를 가리킨다.
사업 정의·사업 준비는 금지 대상이 아니다.

## 2. 문서 상태 (S436 v1.1 신설)

모든 핵심 문서는 첫 줄에 상태를 표기한다. 상태 미표기 문서는 Draft로 간주한다.

| 상태 | 의미 | 근거로 쓸 수 있는가 |
|------|------|------------------|
| Draft | 초안·분석. 작성자 개인 견해 | 안 됨 (참고만) |
| Proposed | 권고안. 승인 대기 | 안 됨 |
| Reviewed | 검토 완료. 미승인 | 안 됨 |
| Approved | 비오님 확정 | 됨 |
| Constitution | 헌법급. 개정 절차 없이 변경 불가 | 됨 (최상위) |
| Superseded | 폐기됨. 후속 문서 반드시 명시 | 안 됨 |

상태가 다른 문서가 충돌하면 상위 상태가 이긴다.
Constitution > Approved > 그 외.
오래된 Approved보다 최신 Approved가 이긴다. 단, 상위 상태를 덮어쓰지는 못한다.

## 3. 판독 상태 용어

| 용어 | 의미 |
|------|------|
| Observed | 문서 일부만 확인한 잠정 상태 |
| Interpreted | 전문 확인 완료. 해석 검증 미실시 |
| Confirmed | 해석 검증까지 완료. 확정 표현 사용 가능 |

확정 표현(Confirmed 전용): 충돌 / 위반 / 무효 / 철회 / 완료 / 정상

## 4. 용어 변경 절차 (S436 v1.1 신설)

용어 정의 변경은 일반 문서 수정이 아니다. 본 문서는 Constitution 등급이다.

변경 요건:
1. 무엇을 어떻게 바꾸는지 구문·신문을 모두 명시한다.
2. 변경이 영향을 주는 기존 문서를 열거한다.
3. 비오님 승인을 받는다. 캐디 단독 변경 금지.
4. 구문을 삭제하지 않고 Superseded로 남긴다.
